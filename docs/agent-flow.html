── 关键设计说明 ──
心跳 vs 主动消息的区别
心跳（路径A）是 LLM 自主决策，可调用 send_message tool 主动发消息；定时任务（路径C）是预定好的机械化发送。两者都是主动消息，都不带 msg_id。
消息排队机制
用户消息到来时若 Agent 正在运行（心跳中），消息进入 messageQueue 而非丢弃，心跳结束后立即处理，保证不漏消息。
被动回复 vs 主动消息
用户发消息后回复（路径B）携带 msg_id，属于被动回复，无需特殊权限，有效期 60 分钟。主动消息（路径A/C）不带 msg_id，每月有配额限制。
API 域名
获取 Token：bots.qq.com
发送消息：api.sgroup.qq.com（v2 接口）
两个域名不能互换，之前 503 的根因。
